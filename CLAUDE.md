### Schema changes (IMPORTANT ŌĆö Turso gotcha)
`prisma db push` / `prisma migrate` do **NOT** work against Turso (`libsql://` ŌåÆ error P1013;
the schema engine only speaks `file:`/postgres). To apply a schema change:
1. Edit `prisma/schema.prisma`, then `npx prisma generate`.
2. Generate DDL: `npx prisma migrate diff --from-empty --to-schema prisma/schema.prisma --script`
   (note: the flag is `--to-schema`, not `--to-schema-datamodel`).
3. Run **only the new** `CREATE TABLE`/`ALTER` statements against Turso via the `@libsql/client`
   `executeMultiple()` (a tiny throwaway script that reads `DATABASE_URL`/`DATABASE_AUTH_TOKEN`
   from `.env`). Inspect data the same way, or with `turso db shell korean-study`.
   (`prisma studio` also can't connect to `libsql://`.)

## Architecture

**Stack:** Next.js 16 App Router, React 19, TypeScript, Tailwind CSS v4, Prisma 7 + libSQL (SQLite/Turso), Claude API (**claude-opus-4-8** extraction with adaptive thinking; **claude-haiku-4-5-20251001** for tap-to-gloss), Google Docs API v1, ElevenLabs / Google Neural2 TTS (`lib/tts.ts` swappable provider; active: ElevenLabs), Vercel Blob (TTS audio cache), lucide-react (nav icons), sharp (icon generation, devDep).

**Data flow:**
1. User triggers sync ŌåÆ `POST /api/sync` fetches the Google Doc via `lib/google-docs.ts` (reads **only the `ņłśņŚģ ļģĖĒŖĖ` tab**; now also captures bold/underline/highlighted spans as an `emphasized[]` list per lesson). Splits into lessons at horizontal rules, hashes each to detect new content. Sends lesson text + emphasized terms + existing deck's normalized fronts to Claude via `lib/extract-cards.ts`. **Exhaustive extraction** ŌĆö every word, grammar pattern, and phrase becomes a card. Upserts cards by `normalizedFront` (real DB dedup). Processes **1 lesson per request** (`MAX_LESSONS_PER_SYNC = 1`) to stay under the 60 s Vercel function timeout. Reports `remaining` so the backlog drains over repeated sync taps. After all upserts, resolves each card's `components[]` lemmas ŌåÆ `CardDependency` edges (two-phase link). For **bulk re-extraction** (all lessons at once), use `npx tsx scripts/local-resync.mts` ŌĆö runs locally with no timeout.
2. Study session ŌåÆ `GET /api/cards/due` returns cards where `CardReview.nextReview <= now`, optionally filtered by `lessonFrom`/`lessonTo` and `scope` (`due` | `ahead`). Fetches the whole eligible pool (up to 1000), runs `selectSessionCards()` to pick a downward-closed set (prerequisites included, cap applied during selection), then `sequenceCards()` from `lib/sequence.ts` to reorder for foundation-first learning. Then queries cards with FSRS state Ōēź 1 once (the "known lemmas" set) and annotates every returned sentence with `unknownCount` via `lib/known-words.ts`. Returns the annotated, ordered list. `StudySession` consumes it **in server order** (no client shuffle). The flashcard Exposure front shows a bare Korean word when the card is new (state 0/1) AND the best sentence still has unknown context words; otherwise shows the sentence directly. When a sentence is shown, the one with fewest unknown words is chosen, tie-broken by `hashStr(id) + reps`. After each answer, `POST /api/review` runs FSRS and updates the schedule.
3. AI practice ŌåÆ `POST /api/generate` sends due cards to Claude via `lib/generate-practice.ts` to create ephemeral extra exercises (not persisted).

**Knowledge graph (2026-06 refactor):** Every card stores `components` (JSON string[] of base-form Hangul lemmas it is *built from* ŌĆö the prerequisites). At sync time these are resolved to `CardDependency` edges. At study time, `GET /api/cards/due` fetches the in-session edges and passes them to `sequenceCards()` (`lib/sequence.ts`), which applies a **blended weighted score**: `score = depth ŌłÆ urgencyBoost`, sort ascending. `depth` = longest in-session prereq chain; `urgencyBoost = min(daysOverdue / 7, 3)`. Foundations come first; a card overdue by 3+ weeks can leapfrog up to 3 prerequisite levels. Cycle-safe (visited-stack DFS). Pure, testable.

**Doc emphasis capture:** `lib/google-docs.ts` now reads `textStyle` (bold / underline / backgroundColor) per text run and returns `{ text, emphasized }[]` per lesson. `emphasized` is a deduped list of the explicitly highlighted spans ŌĆö these are passed to the extraction prompt as a "MUST make a card for each" signal. Body text stays clean (no inline markers) so `targetForm` substring matching is unaffected.

**Exhaustive extraction & dedup:**  `lib/extract-cards.ts` uses `claude-opus-4-8` with `thinking: { type: 'adaptive' }` and streams the response (`.finalMessage()`). Prompt is **exhaustive** ŌĆö every distinct item taught becomes a card (one card per base form; conjugations go in `notes`). Function words get a card only when the lesson explicitly teaches them; otherwise they appear only in `components`. Each card has a new `components: string[]` field. `lib/card-key.ts` exports `normalizeFront(front)` ŌĆö strips English glosses in trailing parens, NFC-normalizes, collapses whitespace ŌĆö used as the DB unique key. Real dedup is enforced by `Card.normalizedFront @unique`; Claude's hint list is a secondary courtesy signal. When editing a card's front, the `[id]/route.ts` update also sets `normalizedFront`.

**Sentence-centric learning:** Cards carry `Sentence[]` ŌĆö 1ŌĆō3 natural Korean example sentences, each storing `korean` (full sentence), `targetForm` (exact surface form to highlight/blank), `translation`, and (at session time) `unknownCount` (number of non-target tokens not yet seen by the learner). Sentences are **presentation context only** ŌĆö `Card` remains the FSRS review unit. `lib/sentence-match.ts` is the single source of truth for substring matching/blanking (handles 1-char/multi-occurrence safety rules). `components/HighlightedSentence.tsx` renders the highlighted span. The **shown sentence** is the one with fewest unknown words (`unknownCount`); ties break via `(hashStr(card.id) + reps) % candidates.length` for variety across reviews. Blank-safety overrides the least-unknown pick when `needsBlank` ŌĆö falls back to the first blank-safe sentence. The flashcard mode has an **Exposure** (sentence shown, or bare word if context not yet learnable) / **Recall** (word blanked) sub-toggle. **Blank-safety:** the first sentence for every card must be blank-safe (targetForm 2+ characters, appears exactly once) ŌĆö the prompt guarantees this.

**Tap-to-gloss (2026-06 P2):** Tapping any word in a Korean example sentence shows a popover with its dictionary form, gloss, and part of speech. Resolution order: (1) corpus-first ŌĆö `normalizeFront(word)` ŌåÆ exact card lookup (instant); (2) stem fallback via `splitParticle`; (3) Setting-table cache (key prefix `gloss:`) for previously-looked-up unknowns; (4) LLM fallback via `claude-haiku-4-5-20251001` (fast, single-word lookups). Cache write is non-blocking. `lib/gloss.ts` owns the prompt + cache helpers. `app/api/gloss/route.ts` is the POST endpoint. `components/GlossProvider.tsx` is a client context mounted once in `app/layout.tsx` ŌĆö exposes `useWordTap()` hook (returns `showGloss(word, anchorRect)` or `undefined` when not mounted); owns the single popover instance and an in-memory request cache (`useRef<Map>`). "Add as card?" in the popover POSTs to `/api/cards`. `components/HighlightedSentence.tsx` accepts an optional `onWordTap` callback; when provided it renders each word token in a `TappableSegment` (inline `<span role="button">`).

**Audio TTS ŌĆö swappable provider (2026-06 P2):** `lib/tts.ts` exports a `TtsProvider` interface and `activeTtsProvider` (chosen by `TTS_PROVIDER` env; default `'google'`). Two providers ship: `googleNeural2Provider` (ko-KR-Neural2-A, reuses `GOOGLE_SERVICE_ACCOUNT_KEY` with `cloud-platform` scope ŌĆö requires the Cloud Text-to-Speech API to be enabled on the GCP project) and `elevenLabsProvider` (`eleven_multilingual_v2`, requires `ELEVENLABS_API_KEY`; currently active). `app/api/tts/route.ts` is a `GET ?text=&voice=` endpoint: hashes `(provider.id, voice, text)` ŌåÆ checks Vercel Blob (`head()`); cache hit ŌåÆ returns stable public URL; miss ŌåÆ synthesizes, `put()` to Blob, returns URL. Reads `KOREAN_BLOB_READ_WRITE_TOKEN` (falls back to `BLOB_READ_WRITE_TOKEN`); returns 503 when unset so the client degrades gracefully. `components/AudioButton.tsx` ŌĆö three-state speaker button (idle/loading/playing); `haptic('selection')` on press; fetches `/api/tts` ŌåÆ `new Audio(url).play()`; falls back to `window.speechSynthesis` (ko-KR, rate 0.9) on non-200. Wired into `StudySession.tsx` on every sentence and every Korean card front, across all three study modes.

**No romanization:** card fronts must be Hangul, never Latin-letter romanization (e.g. `(kkujunhada)`). Short **English clarifying glosses** in parentheses are fine (e.g. `~(ņ£╝)ļĪ£ (direction particle)`). `normalizeFront` strips these glosses when computing the dedup key.

**Database (Prisma + libSQL):**
- `Lesson` ŌåÆ raw doc snapshot per section + contentHash for dedup + **`orderIndex` (Int, 1-based)** for stable lesson numbering. New lessons get `max(orderIndex) + 1` on sync.
- `Card` ŌåÆ front (Korean), back (English), type (vocabulary/grammar/phrase), notes, `distractors` (JSON), **`normalizedFront String @unique`** (dedup key via `normalizeFront()`), **`components String?`** (JSON string[] of prerequisite lemmas). `lessonId` = the lesson where this card was **first introduced** (never overwritten on re-sync). `clozeSentence`/`clozeAnswer`/`clozeTranslation` are **deprecated columns** (kept in DB, no longer written).
- `Sentence` ŌåÆ example sentence attached to a `Card` (CASCADE delete). Fields: `korean`, `targetForm`, `translation`, `orderIndex` (0-based). 1ŌĆō3 per card.
- **`CardDependency`** *(new 2026-06)* ŌåÆ directed prerequisite edge: `cardId` is built from `prerequisiteId`. `@@unique([cardId, prerequisiteId])`. Created at sync time by resolving `Card.components` lemmas ŌåÆ matching `normalizedFront`. Scripts: `apply-graph-ddl.mjs` (one-time DDL), `relink-dependencies.mjs` (full retroactive relink after a corpus-wide resync).
- `CardReview` ŌåÆ FSRS state per card (stability, difficulty, state, reps, lapses, nextReview)
- `StudyDay` ŌåÆ per-day active study time (`date` = user-local habit-day "YYYY-MM-DD", `seconds`, `reviews`)
- `Setting` ŌåÆ generic key/value app settings: `dailyGoalSeconds`, `habitDayStartHour` (int 0ŌĆō23, default 2), `sessionSize` (int 5ŌĆō100, default 20), `readingTextScale` (0.9ŌĆō1.4), `readingAid` ('0'/'1'), `buttonColor` (hex, action accent), `rewardColor` (hex, reward/streak accent). Also stores tap-to-gloss lookup cache under `gloss:<normalizedWord>` prefix (JSON `GlossResult`; non-fatal if missing). NB: the light/dark **theme** is client-only (localStorage), not stored here.

**Habit tracking:** `components/StudySession.tsx` measures active study time and flushes increments to `POST /api/activity` (keyed by the client's **habit-day** date). Use `habitDateStr(hour)` from `lib/habit.ts` wherever you need "today as a habit date"; never use raw `localDateStr()` for activity logging.

**Lesson filtering:** `GET /api/lessons` returns all lessons ordered by `orderIndex`. Both the Cards page and the Study page share `components/LessonRangeFilter.tsx`. Cards page filters client-side; Study page passes `?lessonFrom=&lessonTo=` to `/api/cards/due` server-side. Unassigned cards (`lessonId = null`) are included only under the full "everything" span.

**Study-ahead:** After clearing due cards, the completion screen offers "Study N more ŌåÆ". This calls `GET /api/cards/due?scope=ahead`, which returns up to `sessionSize` not-yet-due cards nearest to becoming due, also sequenced foundation-first. These are real FSRS reviews.

The app uses the **libSQL adapter** (`lib/prisma.ts`), so the same code runs against a local SQLite file in dev and **hosted Turso** in production.

## Scripts (`scripts/`)

One-time / operational scripts ŌĆö run locally against Turso via `@libsql/client` or `npx tsx`:

- `apply-graph-ddl.mjs` ŌĆö Adds `normalizedFront`, `components` columns to `Card` and creates `CardDependency` table. Run ONCE after a clean wipe (UNIQUE index requires an empty table).
- `wipe-card-data.mjs` ŌĆö Deletes all `CardDependency`, `Sentence`, `CardReview`, `Card`, `Lesson` rows. Keeps `StudyDay` and `Setting`. Safe to re-run.
- `local-resync.mts` ŌĆö Full re-extraction of all lessons locally (bypasses 60 s Vercel timeout). Run with `npx tsx scripts/local-resync.mts`. Idempotent ŌĆö skips already-synced lessons by contentHash.
- `relink-dependencies.mjs` ŌĆö Retroactively rebuilds all `CardDependency` edges from stored `components`. Useful after a full resync to catch cross-lesson forward references. Idempotent.
- `find-duplicates.mjs` ŌĆö Scans all cards for near-duplicate fronts using a fuzzy key (strips `~`, all parens). Reports any groups of 2+ for manual review.
- `full-resync.mjs` ŌĆö Drives repeated `POST /api/sync` calls against the live URL until `remaining=0`. Use for incremental syncs; prefer `local-resync.mts` for bulk.
- `apply-sentence-ddl.mjs` ŌĆö One-time DDL that created the `Sentence` table (already applied, do not re-run).
- `add-lesson-order.mjs` ŌĆö One-time backfill that added `orderIndex` to existing lessons (already applied, do not re-run).
- `gen-icons.mjs` ŌĆö Rasterizes `public/icon.svg` (ĒĢ£ mark on brand blue) via `sharp` into the icon set: `icon-192.png`, `icon-512.png`, `apple-icon.png` (180px), `icon-512-maskable.png` (ĒĢ£ scaled to 80% safe zone). Run locally after editing the SVG: `node scripts/gen-icons.mjs`, then commit the PNGs. Requires Apple SD Gothic Neo or another Korean font on the OS.

## Gotchas / conventions

- **Vercel function timeout is 60 s hard limit on Hobby plan** ŌĆö `maxDuration = 300` in the route code has no effect on Hobby. Keep each sync request to 1 lesson. Use `local-resync.mts` for bulk operations.
- **ESLint is strict** (`eslint-config-next` 16). Two rules bite often:
  - `react-hooks/purity` ŌĆö no impure calls during render (`Date.now()`, `Math.random()`, no-arg `new Date()`). Read time/randomness in effects/event handlers, or via a seeded value (see `seededShuffle` and the `seed` memo in `StudySession.tsx`).
  - `react-hooks/set-state-in-effect` ŌĆö don't call `setState` synchronously in an effect body; do it inside async callbacks (`fetch().then(setX)`) or handlers.
- **Lint is clean** (`npm run lint` passes with zero errors). Keep new code clean.
- **Git:** commits are on `main` and auto-deploy to Vercel on push to GitHub.
- **`local-resync.mts` env loading:** it uses dynamic `import()` for lib modules so that `dotenv.config()` runs before Prisma/Anthropic read `process.env`. Static `import` statements are hoisted in ESM and would pick up the wrong `DATABASE_URL`.
- **Vercel CLI:** installed globally at `/opt/homebrew/bin/vercel`. If it's ever missing, `npm i -g vercel`. The `APP_PASSWORD` for the login page is in `.env.local`.
