## Interaction Contract

### RSC-02: Study Page Initial Load

**Before (current behavior):**
- Page mounts in `phase='loading'`, shows inline skeleton (`animate-pulse` divs)
- `useEffect` fires, fetches `/api/lessons` then `/api/cards/due` serially
- Two network round-trips before mode-select screen appears

**After (Phase 11):**
- Page arrives as RSC; `StudyClient` receives `initialCards` (pre-sequenced) and `initialLessons`
- `StudyClient` initializes `phase` as `'select-mode'` (not `'loading'`) when `initialCards.length >= 0` (even 0 is valid — shows empty state)
- Mode-select screen renders on first paint with correct due count
- No `useEffect` fetch on mount; lessons filter and study-ahead still re-fetch via API

**Phase state machine:**

| Phase | Trigger | Visual |
|-------|---------|--------|
| `'select-mode'` | Initial (server-hydrated) or filter re-fetch resolves | Mode-select screen with due count and "Start studying →" |
| `'loading-practice'` | User picks AI practice mode | Pulse skeleton (existing) |
| `'studying'` | Mode selected | `<StudySession>` fullscreen |
| `'complete'` | Session ends | `<SessionComplete>` |

Note: `'loading'` phase is **eliminated** from the initial paint path. It remains only for study-ahead fetch and practice generation (`'loading-practice'`).

### D-06: Filter Re-fetch Spinner

**Trigger:** User changes lesson range in `LessonRangeFilter` → `handleRangeChange` calls `loadDue`.

**Visual behavior:**
1. Mode-select screen stays visible (no phase transition to `'loading'`)
2. `isFilterLoading` state set to `true`
3. The `<p className="text-5xl font-bold" style={{ color: 'var(--reward)' }}>` due-count element is replaced by `<Loader2 className="w-5 h-5 animate-spin text-muted" />` centered in the same vertical slot
4. Filter pill label updates to new range text immediately (optimistic label, confirmed on resolve)
5. On `loadDue` resolve: `setStudyCards(cards)`, `setIsFilterLoading(false)` — count renders with `animate-reveal` fade-in

**Reduced motion:** `animate-spin` is not gated by reduced-motion (spin is functional, not decorative). `animate-reveal` is gated per existing `globals.css` rule.

**Spinner slot size:** The spinner occupies a `h-16` (64px) reserved slot to match the approximate height of the `text-5xl` + caption line pair, preventing layout shift when the spinner swaps to the count.

### D-08 / D-09: Optimistic Grade Advance

**Before (current behavior):**
- `submitReview` is `async`
- `await fetch('/api/review')` blocks before `setQueue` — causes visible ~100–200ms jitter

**After (Phase 11):**
- `submitReview` is synchronous (no `async`)
- Sequence:
  1. Call `reviewCard(current.card.review, rating)` from `lib/fsrs.ts` — pure, instant
  2. Compute `requeue` from the local result
  3. Call `setQueue(nextQueue)` — immediate UI advance
  4. Fire `fetch('/api/review', ...).catch(() => {})` — background, fire-and-forget
- Grade buttons respond instantly; no perceptible delay between tap and card advance
- Haptic fires before the queue update (unchanged from current)

**Failure mode:** If background POST fails — session continues normally, no error shown, no toast. Single-user personal app; acceptable trade-off (D-09).

**Undo:** Snapshot captured before `setQueue` (same timing as today). Undo restores pre-grade visual state. DB may be ahead of visual state after undo if the background POST already succeeded — accepted trade-off (D-10).

### D-11 / D-12: Audio and Flip (No Change)

- Card flip: pure CSS 3D transition (`.card-flip-inner` → `.flipped`). Already instant. No changes.
- Audio (`AudioButton`): idle → loading → playing states. Loading state is acceptable per D-11. No prefetch.

## Accessibility Contract

- Grade buttons retain existing haptic feedback (`haptic('success' | 'selection')`).

- No focus management changes required; Sheet already implements focus trap.
- Reduced-motion: all animation gates already present in `globals.css`; spinner is the only new animated element and is functional (not decorative) — kept in reduced-motion.
