<objective>
Parallelize the two truly-independent Prisma queries in `app/api/cards/due/route.ts` — the card-pool fetch and the known-lemmas fetch — using `Promise.allSettled`, saving roughly one Turso round-trip (~50–100ms) per study-session start (DB-01). The edge-fetch query stays sequential because it structurally depends on the pool card ids. `Promise.allSettled` gives graceful degradation: a rejected critical pool query returns a 500, while a rejected non-critical known-lemmas query degrades to an empty known-set rather than crashing the request.

Purpose: The route currently awaits three queries strictly serially (pool → edges → knownLemmas). The pool and known-lemmas queries have no data dependency on each other and can run concurrently.
Output: A parallelized, graceful-degradation `/api/cards/due` route with identical output semantics.
</objective>
