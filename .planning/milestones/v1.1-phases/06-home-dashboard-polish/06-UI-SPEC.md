## Typography

All values use the existing Tailwind v4 type scale. Four roles are active in the hero:

| Role | Tailwind Class | Size | Weight | Line Height | Usage |

| Display | `text-5xl font-bold` | 48px | 700 | 1 (leading-none) | Due count number (state A) |

**Greeting rule (HOME-02):** The greeting must render at `text-xl font-medium text-foreground` ÔÇö NOT `text-sm text-muted`. This is the primary fix for audit finding F-08.

**Due count rule (HOME-03):** The due number is the visual anchor of the hero. It renders at `text-6xl font-bold leading-none` colored with `text-reward` (var(--reward)). Nothing above it may compete visually.

### Three Hero State Color Contracts

**State A ÔÇö Cards due:**
- Due count: `text-6xl font-bold text-reward leading-none`
- Supporting label: `text-muted text-base`
- CTA: `bg-button text-button-foreground` ÔÇö "Study now ÔåÆ"
- Panel: `bg-surface-1 rounded-2xl shadow-md`

**State B ÔÇö Goal already met today (reviewed AND time ÔëÑ daily goal):**
- Panel background: `bg-reward-soft` tint layered inside `bg-surface-1 rounded-2xl` ÔÇö achieved via an inner div with `bg-reward/10 rounded-xl` wrapping the content, so the card border stays surface-1
- Heading: `text-2xl font-semibold text-foreground` ÔÇö "Goal met today"
- Reward icon: a single large checkmark or ring icon in `text-reward` at `text-4xl`
- Supporting copy: `text-sm text-muted`
- CTA: `bg-button text-button-foreground` ÔÇö "Study ahead ÔåÆ" (still available, lower visual weight ÔÇö full-width button but `text-base` not `text-lg`)

**State C ÔÇö All reviewed, goal not yet met:**
- Heading: `text-xl font-semibold text-foreground` ÔÇö "All caught up"
- Supporting copy: `text-sm text-muted` ÔÇö "Keep going to hit your daily goal."
- CTA: `bg-button text-button-foreground` ÔÇö "Study ahead ÔåÆ"
- Panel: `bg-surface-1 rounded-2xl shadow-md` (same as state A ÔÇö no tint; this is the neutral "good but not done" state)

**Visual distinction rule (HOME-01):** States A, B, and C must be distinguishable at a glance through at least two of: background treatment, heading size/weight, accent color presence, icon presence. Text-only changes are not sufficient.

## Layout & Visual Hierarchy Contract

**Page structure (top to bottom, HOME-03):**

```
[Pull-to-refresh indicator]  ÔåÉ only when active
[Sync feedback message]      ÔåÉ only when present
[Band-up banner]             ÔåÉ only when present
[Hero section]               ÔåÉ ALWAYS visible
  ÔööÔöÇ Greeting (text-xl font-medium text-foreground)
  ÔööÔöÇ Hero content (state-dependent: count OR heading + icon)
  ÔööÔöÇ Primary CTA button (full-width, min-h-14)
[StatsBar]                   ÔåÉ secondary, quiet strip
[HabitTracker]               ÔåÉ habit ring
[ProficiencyArc]             ÔåÉ CEFR progress
[My Korean link card]        ÔåÉ lowest priority
```

**Hierarchy rules:**
1. The greeting breaks from `text-sm text-muted` to `text-xl font-medium text-foreground`. It becomes the section header, not a caption.
2. In state A, the due count (`text-6xl font-bold text-reward`) is the largest element on the page. Nothing above it may use a large typeface.
3. In states B and C, the heading (`text-2xl font-semibold`) is the largest element ÔÇö still smaller than the state A count to preserve relative weight hierarchy across the session.
4. The CTA button stays full-width and below the count/heading in all three states. It never floats or aligns inline.
5. StatsBar sits below the hero with no visual separator ÔÇö the `gap-6` between sections provides enough breathing room.

## Data Requirements

Phase 6 needs to detect whether today's daily goal has been met. This requires calling `/api/activity`:

```
GET /api/activity ÔåÆ { days: [{date, seconds, reviews}], dailyGoalSeconds, dayStartHour }
```

Logic for hero state selection (pure, computed in `useEffect` after both fetches resolve):

```
const todaySeconds = activityData.days.find(d => d.date === habitDateStr(activityData.dayStartHour))?.seconds ?? 0
const goalMet = todaySeconds >= activityData.dailyGoalSeconds && activityData.dailyGoalSeconds > 0

// State A: stats.dueCards > 0
// State B: stats.dueCards === 0 && goalMet
// State C: stats.dueCards === 0 && !goalMet
```

**Purity rule (react-hooks/purity):** `habitDateStr(hour)` must be called inside a `useEffect` or event handler, not during render. The result is stored in state alongside `activityData`.

**ESLint rule (react-hooks/set-state-in-effect):** Both `/api/stats` and `/api/activity` fetch results must flow through `.then(setX)` callbacks, not synchronous `setState` calls in effect bodies.

**Loading state:** While either fetch is pending, the hero renders the existing `<div className="h-28 animate-pulse bg-surface-2 rounded-xl" />` skeleton. No state is shown until both fetches resolve.
