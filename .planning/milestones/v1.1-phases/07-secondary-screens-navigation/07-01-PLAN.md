<objective>
Fix the iOS safe-area-inset-bottom collapse bug (NAV-01) and meet the 44px touch-target minimum on bottom-nav tabs plus add press feedback (SEC-02 / F-01 / F-04).

Purpose: On iPhone the live `env(safe-area-inset-bottom)` reference resolves to `0px` after a Next.js `<Link>` route transition, so the bottom nav loses its home-indicator padding and content sits under the home bar. Reading the value once at mount into a real CSS variable (`--sab`) makes it persist across navigation. The same pass adds the defensive 44px tap height and `active:opacity-70` press feedback the audit flagged as missing.

Output: Updated `components/Nav.tsx` (safe-area useEffect + token swaps + active/min-height classes) and `app/layout.tsx` (`<main>` padding swap).
</objective>

    Inside the `Nav` component body (it is already `'use client'`), add a single `useEffect` with an empty dependency array `[]` that runs once on mount and writes a persistent `--sab` CSS variable to `document.documentElement`. Logic per the UI-SPEC canonical pattern: read the already-set `--sab` from `getComputedStyle(document.documentElement).getPropertyValue('--sab').trim()`; if it is empty, create a temporary `div`, set its inline `style.paddingBottom = 'env(safe-area-inset-bottom)'`, append it to `document.body`, read `getComputedStyle(tmp).paddingBottom`, remove the temp div, then call `document.documentElement.style.setProperty('--sab', value || '0px')`. This reads `env(safe-area-inset-bottom)` while it is still resolvable (at mount, before the first client-side route transition) and freezes it into a real CSS variable so it survives `<Link>` navigation. This is the correct place for a DOM side effect — it satisfies `react-hooks/purity` (the env read happens in an effect, never during render) and `react-hooks/set-state-in-effect` (no `setState` is called).

    On the bottom tab `<nav>` element (currently line 64), change the Tailwind class `pb-[env(safe-area-inset-bottom)]` to `pb-[var(--sab,0px)]`. Leave the rest of that className (`sm:hidden fixed bottom-0 inset-x-0 z-10 bg-surface-1/95 backdrop-blur-md saturate-150 border-t border-border`) unchanged. Do NOT touch the top `<header>` `pt-[env(safe-area-inset-top)]` — only the bottom inset is in scope.
