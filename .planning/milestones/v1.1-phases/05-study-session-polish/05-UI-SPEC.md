## Interaction Contract

### STUDY-01: Progress Indicator

**Requirement:** "Card N of Total" counter visible throughout, updating after each answered card.

**Placement:** Replace the existing `<p className="text-sm text-muted">` at `StudySession.tsx:557–559` with the new format. The ProgressRing stays; only the text label changes.

**Copy:** `Card {stats.reviewed + 1} of {initialCount}`

- `stats.reviewed` = number of cards answered so far (0-based — becomes the 1-based position when +1)
- `initialCount` = total cards at session start (already computed at line 128)
- When `stats.reviewed + 1 > initialCount` (edge case: extra cards requeued), cap display at `initialCount`

**Display format:** `Card 3 of 12` — no period, no punctuation beyond the digits.

**Update timing:** Updates immediately when the queue advances (derived state, no separate update needed).

### STUDY-02: Grade Button Haptics

**Requirement:** Haptically distinct for correct vs. needs-review answers.

**Implementation:** In `submitReview()` at `StudySession.tsx:387`, replace `haptic('selection')` with:

```
haptic(rating >= 3 ? 'success' : 'selection')
```

- `rating >= 3` (Good, Easy): `haptic('success')` — burst pattern `[10, 50, 20]ms`, encouraging
- `rating < 3` (Again, Hard): `haptic('selection')` — single 10ms buzz, softer/neutral

**No visual change to button labels or text.** Haptics fire at the same time as the existing call (before the `await fetch` — this is correct and safe; `rating` is known locally).

### STUDY-03: Korean Typography

**Requirement:** `line-height: 1.7` and `word-break: keep-all` on Korean text.

**Exact edits in `app/globals.css`:**

In `.hangul`: change `line-height: 1.65` to `line-height: 1.7`.
In `.hangul-sentence`: change `line-height: 1.65` to `line-height: 1.7`.

`word-break: keep-all` is already present on both — do not touch it.

No JSX changes needed. The CSS classes are already applied in StudySession.tsx.

### STUDY-04: Inter-Card Transition

**Requirement:** Smooth transition (≤150ms) replacing the abrupt swap when advancing to a new card.

**Animation spec:**
- Class name: `animate-card-in`
- Animation: reuse existing `fadeIn` keyframe (`from { opacity: 0; transform: translateY(4px); } to { opacity: 1; transform: translateY(0); }`)
- Duration: `0.12s` (120ms — within the ≤150ms requirement; slightly snappier than the 150ms grade bar reveal)
- Easing: `ease-out`
- Fill: `forwards`

**CSS to add in `app/globals.css` (after existing `.animate-reveal` block):**

```css
.animate-card-in {
  animation: fadeIn 0.12s ease-out forwards;
}
```

**Reduced-motion gate (add inside `@media (prefers-reduced-motion: reduce)` block):**

```css
.animate-card-in { animation: none; opacity: 1; transform: none; }
```

**JSX:** Wrap the card area (the block that contains both the `mode === 'flashcard'` branch and the non-flashcard branch) with:

```tsx
<div key={cursor} className="animate-card-in w-full">
  {/* existing card content */}
</div>
```

The `key={cursor}` causes React to remount this div on every card advance, triggering the CSS enter animation from scratch. `cursor` is already in scope at the card-rendering site.

**Constraint:** The `key` must go on the outer wrapper, NOT on `.card-flip-container`. The `useLayoutEffect` height measurement is tied to `cursor` and `revealed` and re-runs correctly after remount.

**No conflict with 3D flip:** The flip animation is inside the keyed div, on `.card-flip-inner`. The card always starts `revealed = false` on advance; the flip starts fresh on the new card. The fade-in (120ms) will have completed before the user taps to reveal (flip at 450ms).
