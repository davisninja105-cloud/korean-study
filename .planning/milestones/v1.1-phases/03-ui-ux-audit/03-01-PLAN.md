<audit_reference>
## The 6 Audit Dimensions

| # | Dimension | What to look for |
|---|-----------|------------------|
| 1 | Token escapes | Ad hoc `bg-gray-*`, `text-gray-*`, `border-gray-*`, `bg-white`, `bg-black`, raw `bg-blue-*`/`text-blue-*` used for non-action roles, any literal Tailwind palette color where a semantic token (`--surface-*`, `--muted`, `--border`, `--button`, `--reward`, `--cat-*`) should be used. Blue is reserved for actions — flag non-action blue. |
| 2 | Dark-mode parity | Tokens defined in `:root` (light) but missing a value in the `@media (prefers-color-scheme: dark)` block AND/OR the `:root[data-theme="dark"]` block. Every token needs a value in all 3 blocks. `dark:` Tailwind classes added without a light counterpart, or backgrounds/text added with no `dark:` variant. `--highlight-bg/fg` and `--cat-*` dark values are known suspects. |
| 3 | Spacing & density | Inconsistent padding/gap/sizing between comparable elements; Korean text containers needing more vertical breathing room; scrollable page roots missing bottom padding to clear the fixed bottom nav (`pb-16`/`pb-20`/`pb-24` or safe-area calc). |
| 4 | Interaction feedback | Missing `:active` state on tappable elements; missing hover/disabled states; missing loading states; tap targets below 44x44px (Apple HIG); missing `haptic()` calls on interactions where peers have them. |
| 5 | Typography hierarchy | Heading/body/caption size + weight + color steps that fail to create readable hierarchy; Korean sentence text without `line-height: 1.7` / `word-break: keep-all`; flat hierarchy on the Home hero, flashcard front/back, and HabitTracker card. |
| 6 | Accessibility | Icon-only buttons missing `aria-label`; missing focus management / focus trap; missing `prefers-reduced-motion` counterparts for `@keyframes`; touch targets; color contrast on small badge text; missing `aria-current`. |

## Severity Definitions (D-04 — assign from code evidence, do not pre-assign)

| Level | Meaning | Examples |
|-------|---------|----------|
| P0 | Broken / blocks use | White-on-white in a dark block, invisible text, control with no accessible name and no visible label, element unusable on a device class |
| P1 | Noticeably degraded | Dark-mode token regression (harsh highlight), missing 44px touch target on a primary control, missing `prefers-reduced-motion` on a prominent animation, missing `:active` on a primary control |
| P2 | Polish / nice-to-have | Spacing inconsistency, minor typography refinement, missing haptic on a secondary control |

## Finding Row Format (single Markdown table, one row per finding)

`| ID | Screen/Component | Dimension | Severity | Description | Code evidence |`

- `ID`: stable, sequential — `F-01`, `F-02`, ... Never renumber once assigned.
- `Code evidence`: a concrete `file:line` reference OR a grep result string. Not an assertion — point at real code.

## The 5 Pre-seeded Findings (D-03 — these are confirmed, not hypotheses)

These MUST occupy IDs F-01..F-05 and appear before any code-scan finding. Confirm each against code (read/grep) and attach evidence; assign severity during the read (D-04):

1. Safe-area-inset bug — `env(safe-area-inset-bottom)` resets to 0px after `<Link>` navigation (check `components/Nav.tsx` and any page roots relying on a live `env()` reference)
2. `--highlight-bg/fg` dark-mode regression — no dark values in the dark blocks, so the sentence highlight falls back to the light yellow and renders harsh on dark (check `app/globals.css`)
3. Missing 44px minimum touch targets on some interactive elements (Apple HIG)
4. Missing `:active` state styles on many tappable elements
5. Missing `prefers-reduced-motion` gating on CSS `@keyframes` animations (check `app/globals.css` keyframes vs. its reduced-motion block)
