## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing |
| md | 16px | Default element spacing |
| lg | 24px | Section padding |
| xl | 32px | Layout gaps |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

**Exceptions (Phase 8 specific):**

- Touch target minimum: 44px (Apple HIG) — applies to GlossProvider close button (currently 24px), Undo button (currently ~28px), End button (currently ~28px), and SwipeRow delete button inner height
- GlossProvider "Add as card" affordance: minimum 44px touch target (currently `text-xs` link with no min-h)
- All interactive elements not already meeting 44px must receive `min-h-[44px] min-w-[44px]` or equivalent padding to reach 44px

## Interaction Contract

This section is the core of Phase 8. It specifies the exact changes required for each failing element.

### A11Y-01 — aria-label Audit

Every interactive icon button must have a meaningful `aria-label`. Current state vs. required:

| Component | Element | Current State | Required |
|-----------|---------|--------------|---------|
| `AudioButton` | idle/play button | `aria-label` prop (required, caller-supplied) | Already compliant — callers verified in StudySession.tsx |
| `AudioButton` | loading state | `aria-label="Loading audio…"` | Already compliant |
| `AudioButton` | playing state | `aria-label="Stop audio"` | Already compliant |
| `Nav` | Settings gear (top bar) | `aria-label="Settings"` | Already compliant |
| `Nav` | Bottom tab links (Home/Study/Cards/Habits) | `aria-current` only, no `aria-label` | Add `aria-label={label}` to each tab `<Link>` — the icon+text pair has a text label but screen readers benefit from the explicit label on the link |
| `GlossProvider` | Close button | `aria-label="Close gloss"`, but `w-6 h-6` (24px, F-03) | Compliant label; fix touch target to `min-h-[44px] min-w-[44px]` |
| `GlossProvider` | "Add as card" link | No `aria-label`, no min-h | Add `aria-label="Add {word} as a card"` and `min-h-[44px]` |
| `SwipeRow` | Delete button | `aria-label={deleteLabel}` (prop) | Already compliant |
| `StudySession` | Undo button (↩) | `title="Undo last rating"` only — `title` not reliably read by screen readers (F-12) | Replace `title` with `aria-label="Undo last rating"` |
| `StudySession` | End button | No `aria-label`, text content "End" is an acceptable accessible name | Acceptable as-is (text label suffices); fix touch target (see below) |
| `Sheet` | Dialog container | `aria-label={title}` — undefined when `title` prop omitted (F-26) | Add `aria-label="Study options"` to the mode-selector sheet call site in `app/study/page.tsx`; or add a required `aria-label` fallback in `Sheet.tsx` |

### A11Y-02 — Reduced-Motion Coverage

All `@keyframes` in `globals.css` must have a `prefers-reduced-motion: reduce` counterpart.

**Current state (verified from code):**

| Animation class | Keyframe | Reduced-motion handled |
|----------------|----------|----------------------|
| `.animate-reveal` | `fadeIn` | Yes — `animation: none; opacity: 1; transform: none` |
| `.animate-card-in` | `fadeIn` | Yes — `animation: none; opacity: 1; transform: none` |
| `.animate-burst` | `burst` | Yes — `animation: none; opacity: 0` |
| `.card-flip-inner` | CSS transition | Yes — `transition: none` |
| `.animate-sheet` | `slideUp` → replaced with `fadeBackdrop` | Yes — crossfade instead of slide |
| `.animate-slide-in` | `slideIn` | Yes — `animation: none; opacity: 1; transform: none` |
| `.ring-fill` | `ringFill` | Yes — `animation: none; transition: none` |
| `.animate-backdrop` | `fadeBackdrop` | **NO — missing (F-05)** |

**Required fix:** Add to the `@media (prefers-reduced-motion: reduce)` block in `globals.css`:
```css
.animate-backdrop { animation: none; opacity: 1; }
```

### A11Y-03 — Active States & Tap Highlight

**Global rule:** `-webkit-tap-highlight-color: transparent` must be applied globally (on `html` or `body` in `globals.css`) to suppress the default iOS gray tap flash. This is a single-line addition.

**Per-element active state requirements:**

| Component | Element | Current `:active` | Required |
|-----------|---------|-------------------|---------|
| `Nav` | Bottom tab links | `active:opacity-70` | Already compliant |
| `Nav` | Settings gear | `active:opacity-70` | Already compliant |
| `app/page.tsx` | Study CTA buttons | `active:scale-[0.98] active:opacity-90` | Already compliant |
| `app/page.tsx` | "My Korean" card | `active:shadow-sm active:bg-surface-2` | Already compliant |
| `app/page.tsx` | Band-up dismiss button | No `active:` (F-07) | Add `active:bg-surface-3 active:opacity-80` |
| `AudioButton` | All states | No `active:` | Add `active:bg-button-soft` (idle/play state); `active:opacity-80` (playing state) |
| `GlossProvider` | Close button | No `active:` | Add `active:bg-surface-3` |
| `GlossProvider` | "Add as card" button | No `active:` | Add `active:opacity-70` |
| `StudySession` | Undo button | No `active:` | Add `active:bg-surface-3` |
| `StudySession` | End button | No `active:` | Add `active:bg-surface-3` |
| `components/Sheet.tsx` | Drag handle area | No `active:` needed (passive UI) | None required |
| `app/habits/page.tsx` | "My Korean →" card | `active:shadow-sm active:bg-surface-2` | Already compliant (Phase 7) |

**Touch target fixes (combined with active state pass):**

| Element | Current Size | Required Fix |
|---------|-------------|-------------|
| GlossProvider close button | 24×24px | `min-h-[44px] min-w-[44px]` |
| GlossProvider "Add as card" | ~20px height | `min-h-[44px]`, convert from `<a>` to `<button>` pattern |
| StudySession Undo button | ~28px (`py-1`) | `min-h-[44px] px-3` |
| StudySession End button | ~28px (`py-1`) | `min-h-[44px] px-3` |
