### New Tokens to Add in Phase 4

#### `--muted` тАФ Secondary text color

Purpose: Replace all `text-gray-500 dark:text-gray-400` / `text-gray-400 dark:text-gray-500`
patterns with a single semantic token. This is the most pervasive token escape in the codebase
(~40 instances across 7 screens and 5 components).

| Block | Value | Rationale |
|-------|-------|-----------|
| Light `:root` | `#6b7280` (gray-500 equivalent) | Matches current light-mode usage |
| `@media (prefers-color-scheme: dark) :root` | `#9ca3af` (gray-400 equivalent) | Matches current dark-mode usage |
| `:root[data-theme="dark"]` | `#9ca3af` (gray-400 equivalent) | Must mirror the media-query block |

Registration in `@theme inline`: `--color-muted: var(--muted);`
Tailwind utility generated: `text-muted`, `bg-muted` (bg-muted usage: not expected this phase)

#### `--muted-foreground` тАФ Slightly stronger secondary text

Purpose: Replace `text-gray-600 dark:text-gray-300` (medium-emphasis text, used in labels,
stat values, and list items тАФ ~12 instances). A distinct step above `--muted`.

| Block | Value | Rationale |
|-------|-------|-----------|
| Light `:root` | `#4b5563` (gray-600 equivalent) | Matches current light-mode usage |
| `@media (prefers-color-scheme: dark) :root` | `#d1d5db` (gray-300 equivalent) | Matches current dark-mode usage |
| `:root[data-theme="dark"]` | `#d1d5db` (gray-300 equivalent) | Must mirror the media-query block |

Registration in `@theme inline`: `--color-muted-foreground: var(--muted-foreground);`
Tailwind utility: `text-muted-foreground`

#### `--border` тАФ Default border color

Purpose: Replace `border-gray-200 dark:border-gray-700`, `border-gray-300 dark:border-gray-600`,
and `border-gray-100 dark:border-gray-700` patterns with a semantic border token. Used for
dividers, card edges, form control borders, and separator lines (~15 instances).

| Block | Value | Rationale |
|-------|-------|-----------|
| Light `:root` | `#e5e7eb` (gray-200 equivalent) | Matches most common light border usage |
| `@media (prefers-color-scheme: dark) :root` | `#374151` (gray-700 equivalent) | Matches most common dark border usage |
| `:root[data-theme="dark"]` | `#374151` (gray-700 equivalent) | Must mirror the media-query block |

Registration in `@theme inline`: `--color-border: var(--border);`
Tailwind utility: `border-border`

**Note on stronger borders:** `border-gray-300 dark:border-gray-600` (used on form inputs) is
one step stronger than `--border`. Map these to `border-border` in Phase 4; if visual regression
occurs, the planner may introduce `--border-strong` at `#d1d5db` light / `#4b5563` dark. Decision
left to planner per CONTEXT.md "Claude's Discretion."

### Fixed Token: `--highlight-bg` and `--highlight-fg` (P0 тАФ DS-02)

**Decision D-01 from CONTEXT.md (locked):** Dark-mode highlight uses reversed amber pairing.

| Block | `--highlight-bg` | `--highlight-fg` |
|-------|-----------------|-----------------|
| Light `:root` | `#fde68a` (amber-200, existing) | `#78350f` (amber-900, existing) |
| `@media (prefers-color-scheme: dark) :root` | `#78350f` (amber-900) | `#fde68a` (amber-200) |
| `:root[data-theme="dark"]` | `#78350f` (amber-900) | `#fde68a` (amber-200) |

**Contrast check (D-03):** amber-900 `#78350f` as background against `--surface-1` `#1c2030`
is not a foreground/background pairing тАФ the highlight appears inside a card on `--surface-1`.
The foreground text `#fde68a` (amber-200) on background `#78350f` (amber-900) must achieve тЙе 4.5:1.
Approximate contrast ratio: amber-200 (#fde68a, relative luminance ~0.78) on amber-900 (#78350f,
relative luminance ~0.04) тЙИ 14.5:1. Passes AAA. No nudge needed.

**Component impact:** `components/HighlightedSentence.tsx` already reads `var(--highlight-bg)` and
`var(--highlight-fg)` via inline style тАФ no component changes needed. Fix is globals.css only.

### 60/30/10 Color Distribution (reference, not changed)

| Role | Distribution | Token(s) |
|------|-------------|---------|
| Dominant (60%) | Page background, surface-3 | `--background`, `--surface-3` |
| Secondary (30%) | Cards, sheets, nav, surface-1/2 | `--surface-1`, `--surface-2` |
| Accent (10%) | Action buttons, active nav, focus rings | `--button` |

Accent (`--button`) is reserved for: primary action buttons, active nav tab indicator, focus rings
(`focus:ring-button/50`), hover pill backgrounds (`bg-button-soft`). Never used for decorative
borders, skeleton loaders, or typography.

**`--reward`** (warm orange) is reserved for: streak numbers, ProgressRing fill, goal-met states,
celebration/confetti contexts. Not used for general UI chrome.

*Source: CONTEXT.md locked decisions D-01, D-03; REQUIREMENTS.md DS-01, DS-02, DS-03.*

### Visual regression (light mode must not change)

After the sweep, component appearance in light mode must be visually identical to pre-sweep.
The token values chosen are direct analogs of the gray utilities they replace:
- `text-muted` (#6b7280) = gray-500
- `bg-surface-3` (#f3f4f6) = gray-100
- `bg-surface-1` (#ffffff) = white
- `border-border` (#e5e7eb) = gray-200

### Dark mode correctness (new requirement)

After the sweep, no component should have a text/background pairing that reduces contrast
below WCAG AA (4.5:1 for normal text, 3:1 for large text/icons). The previous state (light
gray utilities with explicit `dark:` variants) was already AA-compliant; the new token values
are direct analogs and maintain the same contrast ratios.
