## Integration Gaps

### NAV-01: Incomplete --sab Adoption (Critical)

**Requirement:** NAV-01 — Bottom navigation correctly applies `env(safe-area-inset-bottom)` via JS-set CSS variable

Phase 7 fixed the Nav and layout `<main>` padding to use the frozen `--sab` CSS variable. However, two out-of-scope files were not updated:

| File | Line | Issue |
|------|------|-------|
| `components/Sheet.tsx` | 81 | `pb-[max(1rem,env(safe-area-inset-bottom))]` — live env() reference |
| `components/StudySession.tsx` | 550 | `pb-[max(1rem,env(safe-area-inset-bottom))]` — live env() reference |

On iPhones with a home indicator, these elements may lose their safe-area padding after client-side `<Link>` navigation. The Sheet is used on every secondary action (mode picker, lesson filter, card editor, settings filter), so this affects the majority of interactive flows.

**Fix (2 lines):**
```tsx
// Sheet.tsx:81 — change
pb-[max(1rem,env(safe-area-inset-bottom))]
// to
pb-[max(1rem,var(--sab,0px))]

// StudySession.tsx:550 — change
pb-[max(1rem,env(safe-area-inset-bottom))]
// to
pb-[max(1rem,var(--sab,0px))]
```

**Confirmed:** `grep -n "env(safe-area-inset-bottom)" components/Sheet.tsx components/StudySession.tsx` returns these two lines.
