## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps inline (gap-1 in `HighlightedSentence` + AudioButton pair) |
| sm | 8px | Compact element spacing (gap-2 in bare-word + AudioButton pair) |
| md | 16px | Default element spacing (gap-4 between card face sections) |
| lg | 24px | Section padding (gap-4 stacks between card sections) |
| xl | 32px | Card face padding (p-8 applied to front and back faces) |
| 2xl | 48px | Not used in this phase |
| 3xl | 64px | Not used in this phase |

Exceptions: The bare-word `<p>` uses `text-5xl` (48px type), which is a typography size not a spacing value. The card face uses `min-h-[220px]` as a height floor — this is an existing constraint, not new.

> Source: Existing `StudySession.tsx` p-8, gap-2, gap-4, gap-1 patterns confirmed at lines 542, 556, 574.

## Typography

This phase adds one new text element (the hint line under the bare word on the front face). All other typography reuses existing patterns unchanged.

| Role | Size | Weight | Line Height | Class Pattern |
|------|------|--------|-------------|---------------|
| Body | 16px (text-base) | 400 (normal) | 1.5 | `text-gray-600 dark:text-gray-300` — card back meaning |

## Component Inventory

All changes are within existing components. No new components are introduced.

### Modified: `components/StudySession.tsx`

**Change 1 — `showBareFront` gate on the front face (PRES-01, PRES-02)**

New derived values (pure, in render — no `Date.now()` / `Math.random()`):

```
isNewCard = !realCard?.review || (realCard.review.state ?? 0) <= 1
showBareFront = mode === 'flashcard' && flashcardSubMode === 'exposure' && isNewCard && cardSentences.length > 0
```

Front face rendering logic at the `chosenSentence ?` branch (line ~546):

- When `showBareFront` is true: render the **bare-word block** (existing pattern at lines 573–582) plus a new hint line beneath it.
- When `showBareFront` is false: existing behavior — sentence block (Exposure with sentence, Recall with blank) or bare-word block (no sentences).

**Exact markup for the new bare-word-front state:**
```tsx
<div className="flex items-center justify-center gap-2">
  <p className="hangul text-5xl font-bold text-gray-800 dark:text-gray-100 text-center">
    {currentCard.front}
  </p>
  <AudioButton
    text={currentCard.front}
    aria-label={`Play: ${currentCard.front}`}
    size="sm"
  />
</div>
<p className="text-xs text-gray-500 dark:text-gray-400 text-center">Recall the meaning</p>
```

The hint text "Recall the meaning" mirrors the existing Recall sub-mode prompt at line 552 ("Recall the missing word") — same class, same position, different copy.

Back face: **unchanged**. The back already shows sentence → divider → Korean word → meaning → notes. The new `showBareFront` state does not alter the back face in any way.

**Change 2 — Least-unknown sentence selection (PRES-03)**

Replaces the `chosenIdx` computation at lines 309–319. Behavior is invisible to the user (no new UI element). The selected sentence may differ from the current rotation-only logic — this is a purely behavioral change.

New type annotation on the client `Sentence` type: `unknownCount?: number` (optional, sourced from server annotation). No visual change.

### New pure module: `lib/known-words.ts`

No UI surface. Pure function, test-covered. No visual contract required.

### Modified: `app/api/cards/due/route.ts`

Server-side annotation of `unknownCount` on each sentence object. No UI surface. Behavioral prerequisite for Change 2 above.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Hint under bare word (new, front face, new card in Exposure mode) | `Recall the meaning` |
| Existing Recall sub-mode hint (unchanged, line 552) | `Recall the missing word` |
| Existing Exposure sentence hint (unchanged, line 570) | `Recall the meaning of the highlighted part` |
| Empty state | Not applicable in this phase — no new screens or empty states |
| Error state | Not applicable in this phase — no new API calls or user-facing error paths |
| Destructive confirmation | Not applicable in this phase |

> "Recall the meaning" is chosen for parity: it is parallel in structure to the existing hints, fits the bare-word cognitive task (produce the English gloss from the Korean word), and has no leading article or imperative "please". Matches CONTEXT.md D-03 discretion note.
