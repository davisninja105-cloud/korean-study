<objective>
Deliver the full user-visible Phase 2 behavior as one vertical slice: the server annotates each example sentence with how many unknown words it contains, and the client (1) introduces brand-new cards as a bare Korean word on the flashcard front in Exposure mode, and (2) picks the least-unknown sentence wherever a sentence is shown. After this plan, a real learner studying a brand-new word sees the bare word first (with a "Recall the meaning" hint) and, on flip, the most readable example sentence — while matured cards and multiple-choice are unchanged.

Purpose: Closes the milestone's "new words are learnable in the moment" promise on the presentation side — prerequisites already come first (Phase 1), and now a new word is shown bare before context, and any sentence shown prefers words already known.

Output: `app/api/cards/due/route.ts` annotating `unknownCount` per sentence, and `components/StudySession.tsx` with the `showBareFront` gate + least-unknown `chosenIdx` selection. Verified by `npm run lint` + `npm run build` clean and a manual dev-session walkthrough.
</objective>
