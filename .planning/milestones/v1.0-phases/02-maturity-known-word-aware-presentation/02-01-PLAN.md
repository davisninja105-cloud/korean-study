<objective>
Build the pure known-word burden helper `countUnknownWords` that the rest of Phase 2 ranks sentences with, plus its unit tests. This is the building block the whole phase slice consumes: the server annotates each sentence with the count this function produces, and the client picks the least-unknown sentence from those counts.

Purpose: A new Korean word should be introduced in the most readable sentence available — one whose *other* words the learner already knows. This helper is the ranking signal that makes "most readable" computable. It is a ranking signal only, never correctness-critical (Korean tokenization is imperfect, accepted in CLAUDE.md re: `splitParticle`).

Output: `lib/known-words.ts` (pure, server+client safe) exporting `countUnknownWords(korean, targetForm, knownLemmas)`, and `tests/known-words.test.ts` covering all five QUAL-02 cases.
</objective>
