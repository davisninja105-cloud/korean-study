# Requirements: Korean Study — Foundation-First Study

**Defined:** 2026-06-25
**Core Value:** When you study, what you're meant to learn is always learnable in the moment — prerequisites come first, and new words are shown bare before context.

## v1 Requirements

Requirements for this milestone. Each maps to roadmap phases.

### Session Selection

- [x] **SEL-01**: A due dependent card never enters a study session without its still-unlearned, in-pool prerequisites also included
- [x] **SEL-02**: Within a session, every card's prerequisites appear before it (the selected set is downward-closed under the prerequisite relation)
- [x] **SEL-03**: Session selection honors `sessionSize` during selection (not only as a post-cap reshuffle), with overshoot bounded by a single prerequisite closure
- [x] **SEL-04**: Prerequisite pull-in is cycle-safe — dependency cycles never cause infinite recursion or duplicate cards
- [x] **SEL-05**: Edge fetching and selection operate over the whole eligible pool (bounded by a safety cap), not only the already-capped most-due IDs

### Presentation

- [ ] **PRES-01**: A brand-new word (no review row, or FSRS state New/Learning) is shown as a bare word on the flashcard front in Exposure mode — not embedded in a sentence
- [ ] **PRES-02**: The back of a new card shows the meaning plus an example sentence as context (with a "recall the meaning" prompt under the bare front)
- [ ] **PRES-03**: When a sentence is shown, the app prefers the sentence with the fewest unknown (not-yet-learned) words; ties broken by the existing rotation
- [ ] **PRES-04**: Existing blank-safety rules are preserved when a sentence must be blanked (Recall / fill-blank)
- [ ] **PRES-05**: Matured cards continue to show the example sentence on the front (no regression); multiple-choice continues to show the bare word

### Quality & Verification

- [x] **QUAL-01**: `selectSessionCards` has unit tests: pool ≤ size returns all; a due dependent pulls in a less-due prerequisite; output is downward-closed and capped; cycle-safe; overshoot bounded by one closure
- [x] **QUAL-02**: `countUnknownWords` has unit tests: counts unknown tokens, excludes the target form, resolves particle stems, respects the known set, all-known → 0
- [ ] **QUAL-03**: `npm run lint` and `npm run build` stay clean; all new sentence/maturity logic is pure in render (no `Date.now()`/`Math.random()` during render)
- [ ] **QUAL-04**: Behavior is confirmed in a real dev study session — a new word appears bare, sentences are readable (least-unknown), and no dependent appears without its still-due prerequisites

## Out of Scope

Explicitly excluded. Documented to prevent scope creep.

| Feature | Reason |
|---------|--------|
| FSRS algorithm changes / new review unit | `Card` stays the review unit; sentences remain presentation-only |
| Perfect Korean tokenization | Known-word counting is a ranking signal only, never correctness-critical (`splitParticle` ambiguity accepted) |
| Pulling in prerequisites outside an active lesson-range filter | Accepted — user scoped to those lessons; out-of-range prereqs stay excluded |
| Bare-word-first in Recall / fill-blank | Those are explicit opt-ins into blanking; they only benefit from least-unknown sentence pick |
