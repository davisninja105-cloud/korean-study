# Milestone v1.1: UI/UX Polish

**Status:** ✅ SHIPPED 2026-06-29
**Phases:** 3–8 (+ 8.1 inserted)
**Total Plans:** 12

## Overview

A systematic visual and interaction polish pass across all screens of the Korean Study app. Starting from a code-only audit that generated 27 severity-classified findings, the milestone executed in dependency order: design system tokens first, then study session, home dashboard, secondary screens, and finally a cross-cutting accessibility pass. A post-completion gap closure phase (8.1) extended the iOS safe-area fix to the two remaining components.

## Milestone Summary

**Key Decisions:**

- Execution order is dependency-driven: Audit → Tokens+Sweep → Study → Home → Secondary+Nav → A11y/PWA (research SUMMARY)
- No new npm packages for the entire milestone — Tailwind v4 CSS keyframes + existing tokens cover all polish
- Strict severity discipline: only P0/P1 (HIGH/MEDIUM) fixed in v1.1; P2 (LOW) deferred to v1.2/backlog
- Every new token must appear in 3 globals.css blocks — validate with grep == 3 hits
- iOS safe-area: freeze `env()` into `--sab` at mount; consume via `var(--sab,0px)` — survives route transitions
- 44px touch target rule does not apply to compact toggle switches — natural h-7 proportion is correct
- `aria-label` preferred over `title` on icon buttons per WCAG F-12

**Issues Resolved:**

- F-02 P0: Dark highlight harsh — reversed amber pairing in both dark blocks (Phase 4)
- F-25 P0: Login focus ring — `focus:ring-button/50` (Phase 4)
- F-23 P0: Wrapped gradient token escape — semantic gradient (Phase 6)
- NAV-01: iOS safe-area collapse after `<Link>` navigation — `--sab` variable (Phase 7 + 8.1)

**Issues Deferred:**

- IN-01: Cards view-toggle `<div>` missing `role="group"` (minor a11y)
- WR-01: `--sab` post-hydration layout shift on first hard-load (inherent to approach)
- NAV-01 runtime behavior unverified on physical iPhone (code wiring correct)
- Phase 3/4 formal VERIFICATION.md not created (low risk)

**Technical Debt Incurred:**

- CR-01: `app/settings/page.tsx` save() doesn't check `res.ok` (pre-existing; now documented)
- CR-02: `app/cards/page.tsx` handleDelete optimistic removal without rollback (pre-existing; now documented)
