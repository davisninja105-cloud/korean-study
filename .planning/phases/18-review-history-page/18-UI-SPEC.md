# Phase 18 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

## Design System

| Property | Value |
|----------|-------|
| Tool | none — this project uses a hand-rolled semantic-token design system (`app/globals.css`), not shadcn. No `components.json` exists anywhere in the repo, and 17 prior phases have consistently extended the manual token system (`--surface-1/2/3`, `--button`, `--reward`, `--cat-*`, `--muted`, `--border`) rather than adopting a component registry. Introducing shadcn now would fork the design system mid-project. This phase reuses the existing manual system exclusively. |
| Preset | not applicable |
| Component library | none (hand-rolled `components/*.tsx` primitives: `Sheet.tsx`, `ProgressRing.tsx`, `HabitHeatmap.tsx`, etc.) |
| Icon library | `lucide-react` (already a dependency; used for nav + `SlidersHorizontal` filter icon on Cards page) |
| Font | `'Pretendard Variable'` (`--font-korean`, self-hosted WOFF2) for Korean text; system `--font-sans` (Geist) for UI chrome/English |

## Spacing Scale

Declared values (must be multiples of 4), matching the scale already in use across `HabitsClient.tsx` / `HabitTracker.tsx`:

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-text gaps, badge internal padding |
| sm | 8px | Compact spacing within a history row (rating pill ↔ timestamp) |
| md | 16px | Default gap between history rows, card internal padding (`p-4`) |
| lg | 24px | Section padding (`p-6`, matching every `HabitsClient.tsx` section) |
| xl | 32px | Gap between major page sections |
| 2xl | 48px | Page top/bottom breathing room |

Exceptions: none. Sentinel/loading-spinner row at the list end uses `py-4` (16px) matching existing empty-state padding conventions (`CardsClient.tsx`).

## Typography

Matches the existing type scale used across `HabitsClient.tsx` / `CardsClient.tsx` — no new sizes introduced.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body (row secondary text — timestamp, filter label) | 12px (`text-xs`) | 400 (regular) | 1.5 |
| Label (row primary — grade name + mastery phrase, card front) | 14px (`text-sm`) | 600 (semibold) | 1.5 |
| Heading (page title "History", section headers) | 20px (`text-xl`)/24px (`text-2xl`) | 600 (semibold) | 1.2 |
| Display (FSRS-state breakdown counts on Habits page) | 24px (`text-2xl`) | 600 (semibold) | 1.2 |

Font weights used: 400 (regular, for body/caption) and 600 (semibold, for all emphasis roles — labels, grade names, headings, and display) — matches the project's existing 2-weight convention (no new weights introduced; every "emphasis" role in `HabitsClient.tsx` collapses to `font-semibold`).

## Color

Per CLAUDE.md's color-system convention: literal Tailwind color utilities are reserved for the app's existing **non-brand semantic exceptions** (red = destructive/"Again", green = success, already used verbatim in `FlashcardMode.tsx`'s grade buttons and `HabitTracker.tsx`'s freeze-comeback pill). All surface/action/reward roles use the semantic CSS tokens — never introduce a new literal color for those roles.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `bg-background` / `bg-surface-1` (white / `#1c2030` dark) | Page background, history row cards, Habits section cards |
| Secondary (30%) | `bg-surface-2` (`#f9fafb` / `#141824` dark) | Habits FSRS-state breakdown tiles (mirrors existing `bg-surface-2` stat tiles), sentinel loading row background |
| Accent (10%) | `text-button` / `bg-button` (`#3b82f6`) | "View history" link text/arrow on Habits stats section only, active "All cards" clear-filter chip text |
| Destructive | `bg-red-50 text-red-500 dark:bg-red-500/10 dark:text-red-400` | "Again" grade rows/badges only — reuses the exact literal token already established in `FlashcardMode.tsx`'s Again button (no new red shade introduced) |

Accent reserved for: the "View history →" link arrow/chevron on the Habits page stats section (D-08), and the active-state text of the "All cards ×" clear-filter chip (D-11) when a card filter is applied. Never used for row backgrounds or grade badges.

**Grade color mapping for history rows (D-02)** — reusing the exact literal tokens `FlashcardMode.tsx` already established for its grade buttons, so the same rating always reads the same color everywhere in the app:

| Rating | Grade name | Badge/row treatment |
|--------|-----------|---------------------|
| 1 (Again) | "Again" | `bg-red-50 text-red-600 dark:bg-red-500/10 dark:text-red-400` — weak, stands out |
| 2 (Hard) | "Hard" | `bg-orange-50 text-orange-600 dark:bg-orange-500/10 dark:text-orange-400` — weak, stands out (distinct from Again so the two "struggling" grades are visually differentiable, per D-02's "weak grades" plural) |
| 3 (Good) | "Good" | `bg-surface-2 text-foreground` — neutral, no color coding (D-03) |
| 4 (Easy) | "Easy" | `bg-surface-2 text-foreground` — neutral, no color coding (D-03) |

Do not reuse `--reward` for Easy rows here — D-03 explicitly keeps Good/Easy neutral in the history feed (unlike the grade *buttons*, which use `--reward` for Easy during live study). The history feed's job is to make struggling cards stand out, not to celebrate easy ones.

## Layout & Component Notes (phase-specific, supplements the generic template)

**`/history` page (`app/history/page.tsx` + `components/HistoryClient.tsx`):**
- Page shell: `<main className="max-w-2xl mx-auto px-4 py-8 flex flex-col gap-6">` — identical container to `HabitsClient.tsx` for visual consistency between the two pages.
- Page header: `<h1 className="text-2xl font-bold text-foreground">History</h1>` + a "← Habits" back-link (`text-sm text-muted hover:text-muted-foreground underline underline-offset-2`), mirroring `HabitsClient.tsx`'s own "← Dashboard" back-link exactly.
- When `?cardId=` is present: render the "All cards ×" chip directly below the header, above the list (D-11).
- Each history row: `bg-surface-1 rounded-xl p-4 flex flex-col gap-1` (a lighter-weight card than the `rounded-2xl shadow-md p-6` sections elsewhere — rows are list items, not standalone sections, so no shadow; a `border border-border` bottom-divider or `divide-y divide-border` on the list wrapper is acceptable instead of per-row cards, whichever reads cleaner at ~20-30 rows/page. Prefer `divide-y` for a denser, more scannable feed).
- Row content, top line: rating pill (grade-colored per the Color section) + card front (Korean, `hangul` class, `text-sm font-medium text-foreground`) truncated with `truncate`.
- Row content, bottom line: `text-xs text-muted` — mastery phrase + timestamp.
- IntersectionObserver sentinel (D-04): an empty `<div ref={sentinelRef} className="h-4" />` at the list end; while fetching the next page, replace it with a centered small spinner (reuse whatever spinner primitive already exists, e.g. the one in `SyncPanel.tsx`, rather than inventing a new one).

**Habits page addition (D-07/D-08):**
- New section inserted between "Averages & consistency" and "Last 30 days" (or immediately after "All-time totals" — planner's call on exact position; keep it high enough to read as a first-class feature, not buried).
- Section wrapper: `bg-surface-1 rounded-2xl shadow-md p-6 flex flex-col gap-3` (identical to every other Habits section) wrapped entirely in a `<Link href="/history">` per D-08 (mirrors `HabitTracker.tsx`'s whole-card-`Link` pattern, `className="...block"`).
- Heading: "Card progress" + a trailing "View history →" affordance (small, `text-xs text-button`, right-aligned next to the heading — do not make the whole heading row look like a separate button; the *entire card* is already the tap target per D-08).
- Body: `grid grid-cols-2 gap-3` of 4 tiles (New / Learning / Review / Relearning), each `bg-surface-2 rounded-xl p-4` with `text-2xl font-bold` count + `text-xs text-muted` label — exact visual clone of the existing "All-time totals" 4-tile grid, just with different data.
- Hover/active states match the section's sibling ("My Korean summary" link): `hover:shadow-lg active:shadow-sm active:bg-surface-2 transition-colors` is NOT applicable here since D-08 wants the *whole card* including the tile grid to be one Link (matching `HabitTracker.tsx`'s pattern, which uses `hover:shadow-lg transition-shadow` on the outer `Link`, not per-tile hover states).

**CardEditor addition (D-10):**
- "View history →" link placed in a sensible existing action-row location within `CardEditor.tsx` (near other card-level actions), styled as a plain text link (`text-sm text-button hover:text-button-hover`), not a button — this is secondary navigation, not a primary action within the editor.
