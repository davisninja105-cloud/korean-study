## Spacing Scale

Declared values (must be multiples of 4) — this is the project's existing scale, confirmed for reuse in Phase 28's new elements (toggle, hint control, anchor copy):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gap inside the hint-reveal pill; inline padding on tappable word spans |
| sm | 8px | Gap between the two toggle segments; gap between hint icon/label and revealed content |
| md | 16px | Default gap inside card faces (`gap-4`, unchanged from `FlashcardMode`) |
| lg | 24px | Section padding around the mode-select toggle block |
| xl | 32px | Card face padding (`p-8`, unchanged from `FlashcardMode`) |
| 2xl | 48px | Not used by this phase's new elements |
| 3xl | 64px | Not used by this phase's new elements |

Exceptions: none. Existing touch-target minimums carry forward unchanged and are already multiples of 4 — reveal CTA `min-h-11` (44px) and grade buttons `min-h-14` (56px). The hint-reveal tap control must also meet the 44px minimum touch target even though its visual pill is compact (pad the tappable area with `py-2 px-3` at minimum, not just the visible icon+text).

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px | 400 (regular) | 1.5 |
| Label | 14px | 600 (semibold) | 1.2 |
| Heading | 24px | *reused 500 (medium) — see note* | 1.4 (English) / 1.7 (Korean, via existing `.hangul-sentence` utility) |
| Caption | 12px | 400 (regular) | 1.4 |

**Note on Heading weight:** the 24px sentence-display role is not new — it is the exact `text-2xl font-medium` treatment `FlashcardMode.tsx` already applies to `chosenSentence` (front) and `displayedSentence` (back). Reuse it verbatim for both the Active English prompt (front) and the Active Korean reveal (back); do not introduce a third weight to "match" the 2-weight rule below — that rule governs the *new* text this phase introduces.

**New elements this phase introduces (must use only Body/Label/Caption above — 2 weights: 400 regular, 600 semibold):**
- Toggle segment labels ("Passive" / "Active") — Label (14px/600)
- Toggle segment descriptors (small subtext under each label, mirrors the existing Exposure/Recall descriptor pattern) — Caption (12px/400)
- Hint-reveal control, collapsed ("Show hint") — Label (14px/600), muted color
- Hint-reveal control, revealed (the `card.back` gloss text) — Body (16px/400), italic, muted-foreground — matches the existing translation/gloss treatment already used elsewhere in `FlashcardMode`
- Reveal instruction line (front, Active only) — Caption (12px/400), muted — matches the existing "Recall the meaning of the highlighted part" precedent
- Grade-anchoring copy (back, Active only, D-09) — Caption (12px/400), muted

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `var(--background)` / `var(--surface-2)` | Page background, mode-select screen backdrop, recessed toggle shell |
| Secondary (30%) | `var(--surface-1)` | Card faces (flip front/back), toggle's selected-segment pill, nav |
| Accent (10%) | `var(--button)` (blue) | Reserved ONLY for: the "Show Answer" primary reveal CTA; the hint-reveal control's icon + label color (signals "tappable, informational"); tap-to-gloss hover/focus rings (existing, unchanged) |
| Destructive | `red-50`/`red-500` (light), `red-500/10`/`red-400` (dark) | The "Again" grade button only — existing, unchanged, reused verbatim in Active mode |

Accent reserved for: the "Show Answer" reveal button, the hint-reveal tap affordance (icon + label only, not its background), and tap-to-gloss word-token focus/hover states. **Explicitly NOT the toggle's selected-segment background** — the Passive/Active toggle must reuse the existing neutral selected-pill convention already established by the Exposure/Recall sub-toggle (`bg-surface-1 text-foreground shadow-sm` on the active segment, `text-muted hover:text-muted-foreground` on the inactive one), not an accent-colored fill. This keeps accent usage restrained and consistent with the rest of the app, where mode/view toggles are neutral and accent is reserved for direct-action buttons.

## Phase 28 Component Notes

Concrete interaction contract for the two new/changed surfaces, supplementing the tables above:

**1. Mode-select toggle** (`ModeSelector.tsx`, replaces the 3-mode grid + Exposure/Recall sub-toggle):
- A single segmented control, full-width within the existing `max-w-sm` container: `flex gap-2 bg-surface-3 rounded-lg p-1` shell (reuses the exact existing sub-toggle shell classes, scaled up as the page's primary control).
- Each segment: `flex-1 py-3 px-4 rounded-md transition-colors` — selected segment gets `bg-surface-1 text-foreground shadow-sm`, unselected gets `text-muted hover:text-muted-foreground`. Label at 14px/600 (Label role), descriptor at 12px/400 (Caption role) directly beneath, matching the existing two-line segment pattern.
- Defaults to "Passive" selected on mount (MODE-02) — no flash of an unselected state.
- The "Include AI-generated practice questions" checkbox stays below the toggle, unchanged in both position and styling (D-04 — available in both modes, no conditional hiding).

**2. Active flashcard face** (`FlashcardMode.tsx`, new front/back branch alongside the existing `showBareFront` / `chosenSentence` / `recallBlanked` branches):
- **Front (not yet revealed):** English translation of `chosenSentence`, styled identically to the existing Korean sentence-front block (24px/medium/centered) but without the `hangul-sentence` class (this is English text) and without a highlight mark (nothing is highlighted pre-reveal). Directly below: the hint-reveal control (collapsed by default), then the reveal-instruction caption ("Translate this, then reveal"). No audio button on the front (there is no Korean text to play yet).
- **Front, new-card degrade (ACTIVE-05):** identical to the existing Passive bare-word/sentence-on-front rendering for that card — reuse the `showBareFront` branch verbatim, no visual distinction from a genuine Passive session.
- **Back (revealed):** Full Korean sentence via `HighlightedSentence` (target form highlighted, tap-to-gloss wired via `onWordTap`, exact same treatment as Passive) plus its `AudioButton`, tap-to-play only per D-01 (no auto-play). Directly below: the grade-anchoring caption. No "See another example →" link (D-02 — hidden entirely in Active; the revealed answer must stay pinned to the sentence the English prompt came from).
- **Hint control:** collapsed state is a small `inline-flex items-center gap-1` pill (`Lightbulb` icon + "Show hint" label, `text-button`, `py-2 px-3` for the 44px touch target) positioned between the English prompt and the reveal-instruction caption. On tap, it expands in place to show `Hint: {card.back}` (Body role, italic, muted-foreground) — no popover/sheet, just an inline reveal (`animate-reveal`, matching the app's existing lightweight reveal transition).
- Grade bar (Again/Hard/Good/Easy) is unchanged — reused verbatim from the existing sticky action bar in `FlashcardMode.tsx`.
