<objective>
Convert `app/cards/page.tsx` from a `'use client'` component that fetches its data in a mount `useEffect` into an async server component (RSC) that fetches cards and lessons directly from Prisma at render time, serializes every `DateTime` field to an ISO string via DTO types, and renders a new `components/CardsClient.tsx` client shell that owns all interactivity. This eliminates the "No cards yet" empty-state flash on first load (RSC-01) and establishes the RSC + client-shell + DTO pattern (RSC-05) that Phases 11 and 12 reuse.

Purpose: The cards page currently hydrates with an empty `cards` state and only populates after a client `useEffect` fetch resolves — producing a visible empty-state flash on first load. Moving the fetch to the server boundary makes the real list arrive with the initial HTML.
Output: An RSC `app/cards/page.tsx`, a new `components/CardsClient.tsx`, and inline DTO types.
</objective>
