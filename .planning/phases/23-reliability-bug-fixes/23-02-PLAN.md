<objective>
Close RELIABILITY-02 and RELIABILITY-03: forward-reference `CardDependency` edges (a card's `components` lemma naming a card taught in a lesson synced LATER) are currently dropped forever unless a human runs `node scripts/relink-dependencies.mjs`. After this plan, every sync that completes with `failed === 0 && newLessons > 0` automatically runs an idempotent full-deck relink pass.

Decisions made during planning (the orchestrator's open questions (a)/(b)/(c), resolved):
- (a) The relink helper lives in a new server-only module `lib/relink-dependencies.ts`, exporting `relinkAllDependencies()`. Its resolution/diff core is a new PURE function `computeMissingEdges()` added to `lib/link-dependencies.ts`, which composes the existing `resolveDependencyEdges()` — honoring that file's IN-02 mandate that resolution logic is never reimplemented.
- (b) `runSync()` in `lib/sync.ts` calls the helper directly, just before building its return value. Both `app/api/sync/route.ts` and `app/api/cron/sync/route.ts` call `runSync` (verified: cron route line 23), so both inherit the behavior with zero route changes.
- (c) `scripts/relink-dependencies.mjs` is DELETED and replaced by a thin `scripts/relink-dependencies.mts` wrapper that calls the shared helper (env-first dynamic-import pattern from `scripts/local-resync.mts`). The old `.mjs` hand-rolls its own `normalizeFront` copy AND builds its lookup only from cards `WHERE components IS NOT NULL` — omitting leaf-prerequisite cards, the exact CR-02 bug shape — so keeping it as-is would preserve a divergent, subtly-wrong implementation. Additionally, the final relink pass in `scripts/local-resync.mts` (lines ~211-239) is the third copy of the full-deck relink persistence loop; it is consolidated onto the same helper.

Purpose: the knowledge graph's foundation-first promise depends on complete prerequisite edges; today they silently rot as new lessons land.
Output: pure resolver-diff function + server relink helper + sync hook + real-DB idempotency regression test + script consolidation + CLAUDE.md updates.
</objective>
