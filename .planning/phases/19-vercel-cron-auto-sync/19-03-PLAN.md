<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Vercel Cron scheduler → GET /api/cron/sync | Vercel auto-attaches `Authorization: Bearer <CRON_SECRET>`; the app must verify it before running the sync. |
| Public internet → GET /api/cron/sync | Any unauthenticated caller reaching the cron path must be rejected (401) — the path stays inside the auth matcher. |
| app → Claude/Google APIs (via runSync) | An authorized cron run drives paid Claude extraction calls; abuse potential if the secret leaks. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-19-01 | Elevation of Privilege | middleware.ts / isValidCronAuth (fail-open when CRON_SECRET unset) | high | mitigate | Fail-closed predicate: true only when cronSecret is a non-empty string AND the header exactly matches the Bearer-prefixed secret; expressed as one conjunction so a missing/empty secret never validates. `tests/auth.test.ts` asserts the secret-unset and secret-empty cases return false. |
| T-19-03 | Tampering / Information Disclosure | middleware.ts matcher (cron path carved out → publicly reachable) | high | mitigate | The cron path stays inside the existing negative-lookahead matcher; the matcher is left byte-for-byte unchanged and Task 2's acceptance criterion verifies /api/cron/sync is NOT added to the exclusion list (Success Criterion 2). |
| T-19-02 | Spoofing / Denial of Service | GET /api/cron/sync (leaked/guessed CRON_SECRET → repeated paid Claude sync runs) | medium | accept | Rate limiting is explicitly OUT OF SCOPE per .planning/REQUIREMENTS.md (single-tenant, Hobby timeout caps damage). The >=16-char random CRON_SECRET is the mitigation — same posture as the existing shared-password auth. |
| T-19-06 | Information Disclosure | app/api/cron/sync error responses | low | mitigate | Catch logs server-side (console.error) and returns a generic "Cron sync failed — see server logs" message; no internal error detail leaked to the caller (matches the app's API-route error convention). |
| T-19-SC | Tampering | npm/pip/cargo installs | low | accept | Zero new dependencies this phase (verified in RESEARCH Package Legitimacy Audit); no install task exists. |
</threat_model>
