## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Live production DB → local script | The script reads the real Turso deck holding irreplaceable FSRS review history; any write would be unrecoverable data damage |
| Stored DB JSON columns → parser | `components`/`distractors` are untrusted legacy JSON (malformed rows exist in production) |
| Rendered report → git history | `.planning/audits/*.md` is committed and pushed — anything interpolated into it becomes permanent public-ish record |

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |

| T-21-01 | Tampering | scripts/audit-cards.mts Prisma usage | high | mitigate | Single findMany read is the only DB access; structural grep for write-capable Prisma calls is a blocking acceptance criterion in BOTH tasks; lib/audit-checks.ts has no Prisma import at all (gated in Plan 21-01) |
| T-21-02 | Information Disclosure | Markdown report committed to git | medium | mitigate | Renderer interpolates only AuditFindings fields + the date — never process.env or the database URL; Task 2 gates the report file on containing no libsql connection-string material |
