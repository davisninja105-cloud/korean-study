<objective>
Convert the habits page from a `'use client'` page that mounts in a `days===null` empty state and runs two `useEffect` fetches, to an async RSC that server-fetches activity + masteredCount (D-03) and hands them to a new `HabitsClient` shell that starts populated (RSC-04 — no empty heatmap flash). The habits page has no pull-to-refresh and no data-mutating interactivity, so all data arrives via RSC props and no client refetch occurs (D-04). The only permitted loading is the one-tick `HabitsLoading` skeleton while client-local `today` resolves (D-02).

Purpose: Eliminate the empty-heatmap flash on first load of `/habits` by moving data acquisition to the server render. The Vercel server runs UTC and cannot compute the habit-day `today` (which can roll at 2am local per `dayStartHour`), so `today` stays client-side per D-01; the one-tick skeleton is the intentional loading state, not an empty-state flash.
Output: `components/HabitsClient.tsx` (new client shell), `app/habits/page.tsx` (async RSC).
</objective>
