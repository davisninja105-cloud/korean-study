<objective>
Add the `ReviewLog` Prisma model and apply it to the live Turso database via the project's manual-DDL workaround, so an append-only, idempotency-keyed audit table exists before any write path references it.

Purpose: This is the schema-gated foundation for the phase (HIST-01 rows land in this table; HIST-02 idempotency is enforced by its UNIQUE index). Without the live table, build/type checks would pass on the schema file alone while every runtime write 500s — the exact false-positive the schema-push gate exists to prevent.
Output: `model ReviewLog` in `prisma/schema.prisma`, a generated Prisma client exposing `prisma.reviewLog`, a new `scripts/apply-reviewlog-ddl.mjs`, and the live `ReviewLog` table + indexes in Turso.
</objective>
