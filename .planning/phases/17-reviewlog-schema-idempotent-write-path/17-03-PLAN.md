<objective>
Thread a client-generated idempotency key through the review retry chain and add undo-triggered cancellation, so the browser half of the idempotent write path is complete.

Purpose: Delivers the client half of HIST-02 (one stable key reused across retries → the server dedups correctly) and all of HIST-03 (undo aborts the in-flight retry chain so a stale attempt can never re-apply a rating after undo). Per RESEARCH Pitfall 4, cancellation is a separate, necessary mechanism — the idempotency key alone does not cover a first-ever-successful delivery that arrives after undo.
Output: A modified `components/StudySession.tsx` where `submitReview` generates a per-grade key + AbortController, `postReviewWithRetry` reuses the key and honors a cancel signal, `undoRef` carries the controller, and `handleUndo` aborts it.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user undo action → in-flight background retry | An undo must cancel a not-yet-delivered retry so a stale rating cannot re-apply after the review is reverted |
| client → POST /api/review | The client generates the idempotency key that the server (17-02) uses to dedup |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-17-02 | Tampering | Stale background retry re-applying a rating after undo | high | mitigate | Per-grade `AbortController` stored on the single-slot `undoRef.current`; `handleUndo` calls `controller.abort()` before the retry chain's next attempt, and `postReviewWithRetry` checks `cancelSignal.aborted` at loop-top, in catch, and after each backoff wait. Best-effort for an already-in-flight request (RESEARCH Open Question 1 — narrow, accepted residual window); fully covers the dominant multi-second backoff window. |
| T-17-01 | Tampering | Per-attempt key generation defeating server-side dedup | high | mitigate | Key generated exactly ONCE in submitReview and passed unchanged into postReviewWithRetry — never regenerated per attempt (RESEARCH anti-pattern) |
| T-17-06 | Tampering | Impure key/controller generation breaking render (react-hooks/purity) | low | mitigate | `crypto.randomUUID()` / `new AbortController()` stay in the event-handler body; lint gate enforces purity (Pitfall 5) |
| T-17-SC | Tampering | npm/pip/cargo installs | n/a | accept | No package installs this phase — uses only platform built-ins (`crypto.randomUUID`, `AbortController`) already available in the browser. Supply-chain vector N/A. |
</threat_model>
